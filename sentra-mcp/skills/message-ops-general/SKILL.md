# 通用消息操作（Message Ops）

## 通用工作流
1. 明确目标对象
   - 哪个平台/哪个群/哪个会话/哪个用户
2. 若涉及“操作某条消息”（撤回/引用/点赞等）
   - 必须先拿到 message_id（从上下文或通过 list/search 获取）
3. 执行动作
4. 结果确认
   - 如果返回 success=false 或错误码，输出可恢复的下一步（例如重新获取 message_id、重新鉴权、稍后重试）

## 参数注意事项
- id/path/url 必须来自上下文或工具返回
- 如果用户给了多个候选（例如多个群名/多个会话），优先让系统先 list/search 再选择

## 输出偏好
- 执行后返回可验证的结果（例如 id、数量、摘要）
