# QQ 账号 / 系统 / 用户操作

## 适用场景
- 账号资料与头像：查看/修改个人资料、头像、个性签名（longnick）。
- 系统状态：查看/设置在线状态、模型展示。
- 用户互动：点赞、戳一戳、查看资料卡点赞信息、删除好友。

## 工作流
1. 默认先查再改
   - 查询资料/状态：`*_get*`
   - 修改资料/状态：`*_set*`
2. 执行动作要可验证
   - 输出简短回执：成功/失败 + 关键字段（如目标 user_id）。

## 参数/约束
- 涉及 `user_id`：必须来自上下文或工具结果。
- 修改类操作属于真实动作，避免“多余解释/角色扮演”。
